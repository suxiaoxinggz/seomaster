我直接用你截图里的几个“主流可免费用”的翻译提供商来整理，一家一家把 免费额度 + 主要翻译接口 URL + 请求体字段 + 返回体字段（含类型） 写清楚，方便你做集成或在你的软件里配置。

⚠️说明：下面说的“免费”基本都是 有永久免费额度/Free Tier，超出额度就按量计费，具体价格以后可能调整，要以官网为准。

⸻

一、DeepL API Free

1. 免费额度
	•	DeepL API Free：每月可免费翻译 500,000 字符，超出需升级 Pro 付费。 ￼

2. 基本信息
	•	基础 URL（文本翻译）
	•	Free 计划：https://api-free.deepl.com/v2/translate  ￼
	•	HTTP 方法：POST
	•	认证方式：HTTP 头
	•	Authorization: DeepL-Auth-Key {YOUR_API_KEY}  ￼
	•	Content-Type：application/json（也支持 x-www-form-urlencoded，这里只写 JSON）

3. 请求体主要字段（JSON）

来自官方 “Translate Text” 文档 ￼

字段名	类型	必填	说明
text	array<string>	✅	要翻译的文本数组（最多 50 条）。返回顺序与此数组顺序一致。
target_lang	string	✅	目标语言代码，如 EN, DE, ZH 等。
source_lang	string	可选	源语言代码，省略则自动检测。
enable_beta_languages	boolean	可选	是否启用额外 Beta 语言。
context	string	可选	给翻译模型的上下文提示，不会被翻译，也不计费。
model_type	string (枚举)	可选	latency_optimized（默认，延迟低）或 quality_optimized（质量高）。
split_sentences	string/number	可选	句子切分策略，例如 0, 1, nonewlines。
preserve_formatting	boolean/number	可选	true 时更保留原格式。
formality	string	可选	目标语言支持时可用，如 default, more, less。
glossary_id	string	可选	术语表 ID，需同时指定 source_lang 和 target_lang。 ￼

对于简单使用，其实只用 text + target_lang（可选 source_lang）就够了。

4. 返回体字段（JSON）

示例： ￼

{
  "translations": [
    {
      "detected_source_language": "EN",
      "text": "Hallo, Welt!"
    }
  ]
}

字段说明：

字段名	类型	说明
translations	array<object>	翻译结果数组，与请求中的 text 对应。
translations[].detected_source_language	string	自动检测到的源语言代码（如果未在请求中指定 source_lang）。
translations[].text	string	翻译后的文本。

二、Google Cloud Translation API v3

1. 免费额度
	•	Cloud Translation Basic + Advanced：每月前 500,000 字符免费，无过期时间（不包括 LLM 模式）。 ￼

2. 基本信息
	•	主要文本翻译接口（v3 Advanced）：
POST https://translate.googleapis.com/v3/projects/{PROJECT_ID}/locations/{LOCATION}:translateText  ￼
	•	认证：推荐使用 Service Account/OAuth2，请求需带 OAuth token（Authorization: Bearer {token}），也可用 key=API_KEY 查询参数（安全性略低）。 ￼
	•	Content-Type：application/json

3. 请求体字段（TranslateTextRequest）

官方 JSON 结构： ￼

{
  "contents": ["string"],
  "mimeType": "string",
  "sourceLanguageCode": "string",
  "targetLanguageCode": "string",
  "model": "string",
  "glossaryConfig": { /* object */ },
  "transliterationConfig": { /* object */ },
  "labels": { "key": "value" }
}

字段明细：

字段名	类型	必填	说明
contents	array<string>	✅	要翻译的文本数组，推荐总长度 < 30,000 codepoints。
mimeType	string	可选	源文本格式，默认 "text/html"，常用 "text/plain"。
sourceLanguageCode	string	可选	源语言代码，如 en-US，省略则自动检测。
targetLanguageCode	string	✅	目标语言代码，如 zh-CN。
model	string	可选	使用的模型路径，如 NMT / Translation LLM / AutoML。
glossaryConfig	object	可选	术语表配置，需与 location 区域一致。
transliterationConfig	object	可选	转写配置。
labels	map<string,string>	可选	自定义标签，用于计费/审计。

4. 返回体字段（TranslateTextResponse）

结构： ￼

{
  "translations": [ { /* Translation */ } ],
  "glossaryTranslations": [ { /* Translation */ } ]
}

Translation 对象： ￼

字段名	类型	说明
translatedText	string	目标语言文本。
model	string	实际使用的模型路径（规范化后）。
detectedLanguageCode	string	自动检测的源语言代码（未显式指定 sourceLanguageCode 时）。
glossaryConfig	object	本次翻译使用的 glossary 配置。

	•	translations：正常翻译结果数组，长度与 contents 相同。
	•	glossaryTranslations：如果启用了术语表，这里是应用术语表后的结果；否则可能等同于 translations。 ￼

三、Microsoft Azure Translator（Text Translation API v3）

1. 免费额度
	•	Azure 免费服务页面写明：Translator 每月 2,000,000 字符免费额度（Free Tier，一直有效）。 ￼
	•	Q&A 文档也说明 Free tier 每月 2M 字符的限制。 ￼

2. 基本信息
	•	基础 URL（全局）：
https://api.cognitive.microsofttranslator.com/translate?api-version=3.0  ￼
	•	可以再加 &to=xx（目标语言），&from=yy（源语言，可选）等查询参数。
	•	认证头： ￼

Header	说明
Ocp-Apim-Subscription-Key	必填，你的 Translator 资源密钥。
Ocp-Apim-Subscription-Region	区域资源时必填，值为资源所在 region（如 eastasia）。
Content-Type	application/json; charset=UTF-8



3. 请求体字段

请求体是 JSON 数组，数组每个元素是一个对象： ￼

[
  { "text": "Hello, what is your name?" }
]

每个元素字段：

字段名	类型	必填	说明
text	string	✅	要翻译的文本。

常用查询参数（URL 上） ￼

参数	类型	必填	说明
api-version	string	✅	固定 3.0。
to	string / 多个	✅	目标语言代码，可重复多次表示多目标语言，如 to=de&to=zh-Hans。
from	string	可选	源语言代码，省略则自动检测。
textType	string	可选	plain 或 html（默认 plain）。
category	string	可选	自定义模型分类（Custom Translator）。
…			还有 profanity / alignment / sentenceLength 等可选参数。

4. 返回体字段

返回体同样是 JSON 数组，每个元素对应请求体中的一个文本： ￼

[
  {
    "detectedLanguage": {
      "language": "en",
      "score": 1.0
    },
    "translations": [
      { "text": "你好, 你叫什么名字？", "to": "zh-Hans" }
    ]
  }
]

字段说明：

字段名	类型	说明
detectedLanguage	object	自动检测到的源语言信息（如果未指定 from）。
detectedLanguage.language	string	检测的语言代码，如 en。
detectedLanguage.score	number (float)	置信度 0–1。
sourceText	object	若请求文本不是该语言默认脚本，会返回标准脚本形式：sourceText.text。
translations	array<object>	译文数组（一个 to 就一项，多 to 就多项）。
translations[].text	string	翻译后的文本。
translations[].to	string	该译文对应的目标语言代码。

四、LibreTranslate（开源、可自建）

1. 免费情况
	•	官方宣传是 “Free and Open Source Machine Translation API”，你可以： ￼
	•	直接用公开实例 https://libretranslate.com（有速率/配额限制，需申请 api_key）；
	•	或者完全自建服务器（自托管时通常不需要 API key，严格来说就是真正“免费翻译”）。 ￼

2. 基本信息
	•	文本翻译接口：
	•	POST https://libretranslate.com/translate（或你自己的部署地址 /translate） ￼
	•	Content-Type：application/json 或 application/x-www-form-urlencoded（文档用的是 JSON）。

3. 请求体字段

官方 API Usage + API Reference： ￼

{
  "q": "Hello!",
  "source": "en",
  "target": "es",
  "format": "text",
  "alternatives": 3,
  "api_key": "xxxxxx"
}

字段说明：

字段名	类型	必填	说明
q	string 或 array<string>	✅	要翻译的文本，支持数组批量。
source	string	✅	源语言代码，或 "auto" 自动检测。
target	string	✅	目标语言代码。
format	string	可选	text（默认）或 html。
alternatives	integer	可选	希望返回的备选翻译数量。
api_key	string	视实例配置	公共实例通常必填；自建默认不需要。

4. 返回体字段

简单翻译示例返回： ￼

{
  "translatedText": "¡Hola!"
}

启用自动检测时： ￼

{
  "detectedLanguage": {
    "confidence": 90.0,
    "language": "fr"
  },
  "translatedText": "Hello!"
}

启用 alternatives 时： ￼

{
  "alternatives": ["Salve", "Pronto"],
  "translatedText": "Ciao"
}

字段说明：

字段名	类型	说明
translatedText	string 或 array<string>	主翻译结果。
detectedLanguage	object	自动检测的源语言信息。
detectedLanguage.language	string	语言代码。
detectedLanguage.confidence	number	置信度（0–100）。
alternatives	array<string> 或 array<array<string>>	备选翻译。
